# Chinese Checkers / 跳棋游戏

一个基于 **C++17、Visual Studio 2022 和 EasyX 图形库** 编写的图形化跳棋小游戏。项目使用标准跳棋星形棋盘结构，支持 2–6 名玩家参与，支持每人 3、6 或 10 个棋子，并实现了棋子选择、合法落点提示、一步移动、连续跳跃、胜利判断、返回上一步、重新开始、键盘编号操作和 UTF-8 日志记录等功能。项目适合作为 C++ 课程实践、图形化界面练习、棋盘游戏规则建模和搜索算法应用示例。

## 2. 项目简介

本项目实现了一个本地运行的图形化跳棋游戏。程序启动后，玩家可以在同一设置界面调整参加人数和每人棋子数量，也可以直接使用默认设置开始游戏。进入游戏后，系统按玩家编号顺序行动。当前玩家可以用鼠标点击自己的棋子，也可以按棋子上显示的数字编号选择棋子；程序会自动计算并标记该棋子的所有合法落点，玩家再点击绿色标记或按落点编号即可完成移动。

棋盘采用标准跳棋星形结构，共 17 行、121 个孔位。每名玩家从自己的起始营地出发，目标是将全部棋子移动到正对面的停车区。项目通过几何坐标生成棋盘孔位，通过距离判断生成相邻边和跳跃边，并使用搜索算法计算连续跳跃落点。

整体代码拆分为棋盘逻辑、游戏流程、绘图界面、键盘操作、数据结构和日志系统几个模块，便于阅读、维护和课程答辩说明。相比旧版，本版没有改变核心棋盘算法，而是在交互体验、2 人模式和设置流程上进行了补充。

## 3. 已实现功能

### 3.1 游戏设置

- 支持选择 2、3、4、5、6 名玩家。
- 支持选择每名玩家 3、6、10 个棋子。
- 初始默认设置为 3 人、每人 3 个棋子。
- 设置界面支持通过按钮减少或增加玩家人数。
- 设置界面支持通过按钮选择每人棋子数量。
- 设置界面支持键盘方向键调整：左右方向键调整人数，上下方向键调整棋子数量。
- 支持按回车键直接按照当前设置开始游戏。
- 根据玩家人数自动分配起始营地。
- 每名玩家的目标停车区为起始营地正对面的三角区。
- 支持红、蓝、绿、橙、紫、青六种棋子颜色。

### 3.2 棋盘系统

- 使用标准跳棋星形棋盘。
- 棋盘总行数为 17 行。
- 每行孔位数量为：

```text
1, 2, 3, 4, 13, 12, 11, 10, 9, 10, 11, 12, 13, 4, 3, 2, 1
```

- 总孔位数为 121。
- 自动生成每个孔位的屏幕坐标。
- 自动生成相邻移动边。
- 自动生成跳跃边。
- 自动识别鼠标点击到的孔位。
- 新版补充了 2 人模式的对称营地分配，2 人对局时使用上方营地和下方营地。

### 3.3 棋子移动规则

当前版本支持两类移动：

1. **一步移动**  
   如果相邻孔位为空，棋子可以移动到该孔位。

2. **跳跃移动**  
   如果相邻方向上存在一个被跳过的棋子，并且该方向后方落点为空，则可以跳到该落点。

连续跳跃通过搜索算法计算。玩家点击棋子或按数字键选择棋子后，程序会一次性给出所有可落点，不需要玩家手动逐步试探。可落点既可以使用鼠标点击，也可以使用键盘编号选择。

### 3.4 连续跳跃搜索

- 支持一次行动内连续跳跃。
- 使用队列搜索所有可达跳跃落点。
- 使用 `visited` 集合避免重复访问和死循环。
- 使用虚拟局面判断连续跳跃过程中的当前位置、起点和其他棋子占用情况。
- 所有合法落点会以绿色圆圈显示在棋盘上。
- 前 9 个合法落点会显示键盘编号，方便直接按数字键落子。

### 3.5 回合与胜利判断

- 当前版本行动顺序固定为玩家编号从小到大。
- 每次成功移动后，自动切换到下一名玩家。
- 每名玩家右侧显示完成进度。
- 当某名玩家的全部棋子进入目标停车区时，游戏结束。
- 游戏结束后显示获胜玩家，并支持重新开始或返回上一步。
- 鼠标移动和键盘移动共用同一套胜利判断逻辑，避免不同输入方式造成规则不一致。

### 3.6 图形界面功能

- 使用 EasyX 创建图形窗口。
- 窗口大小为 1120 × 760。
- 左侧为棋盘区域，右侧为信息栏。
- 右侧信息栏显示当前阶段、当前玩家、操作提示和玩家进度。
- 设置界面显示人数加减按钮、棋子数量按钮、开始游戏按钮和退出按钮。
- 棋子使用不同颜色区分玩家。
- 当前选中棋子使用加粗边框显示。
- 当前玩家目标停车区会以浅绿色高亮。
- 可落点使用绿色圆圈标记。
- 当前玩家棋子上会显示键盘选择编号。
- 可落点上会显示键盘落点编号。
- 右侧信息栏会显示当前棋子编号列表或可走位置编号列表。
- 支持鼠标点击完成所有主要操作，也支持键盘完成设置、选子、落子和返回。

### 3.7 返回与重新开始

- 设置阶段可以直接调整人数和棋子数量，不需要分两步进入不同界面。
- 选择落点阶段可以取消当前棋子选择。
- 游戏进行阶段可以撤销上一次已经完成的移动。
- 游戏结束阶段也可以撤销最近一步移动。
- 支持重新开始游戏。
- 支持 `Esc` 快捷键执行返回上一步。
- 支持 `Delete` 快捷键执行返回上一步。
- 在选择落点阶段，按数字 `0` 可以取消当前棋子选择并返回选棋阶段。

### 3.8 日志系统

- 程序启动时自动创建日志文件。
- 日志目录为 `result/`。
- 日志文件名格式为：`YYYYMMDD_HHMMSS.txt`。
- 日志使用 UTF-8 with BOM 编码，中文内容可直接阅读。
- 记录内容包括：程序启动、人数调整、棋子数量调整、键盘开始游戏、玩家营地分配、行动顺序、棋子选择、棋子移动、撤销操作、无效点击、无效按键和胜利结果等。

1. 下载或克隆本项目到本地。
2. 使用 Visual Studio 2022 打开解决方案文件：

```text
ChineseCheckers.sln
```


```text
Release | x64
```

### 7.2 `game.cpp`

`game.cpp` 是游戏流程核心，主要负责：

- 初始化游戏状态。
- 处理人数选择和棋子数量选择。
- 创建玩家和棋子。
- 分配起点营地和目标停车区。
- 生成行动顺序。
- 处理鼠标点击事件。
- 处理键盘按键和字符消息。
- 在设置阶段处理方向键和回车键。
- 在走棋阶段把数字键操作分发给键盘模块。
- 切换游戏阶段。
- 处理棋子选择和落点选择。
- 保存移动历史。
- 实现返回上一步。
- 判断玩家是否获胜。
- 控制 EasyX 主循环。

`board.cpp` 是棋盘规则核心，主要负责：

- 根据两个孔位之间的中点生成跳跃关系。

- 根据棋子数量返回三角区孔位。
- 根据鼠标坐标查找孔位。
- 查找指定孔位上的棋子。
- 计算一个棋子的全部合法落点。
- 统计玩家进入停车区的棋子数量。
- 判断玩家是否已经完成全部棋子进区。

### 7.4 `drawer.cpp`

`drawer.cpp` 是图形界面层，主要负责：

`gamedata.cpp` 主要负责把枚举值转换成中文显示文本，例如游戏阶段名称和营地名称。新版在按钮类型中补充了人数减少、人数增加和开始游戏等操作编号，但显示文本转换方式仍保持旧版风格。

## 8. 主要算法与实现思路

### 8.1 游戏状态机

项目使用 `phasekind` 枚举管理游戏阶段：

```cpp
enum phasekind
{
    phase_player_count,
    phase_piece_count,
    phase_select_piece,
    phase_select_target,
    phase_game_over
};
```

不同阶段只允许执行对应操作。例如设置阶段负责调整人数、棋子数量和开始游戏；选棋阶段只能选择当前玩家自己的棋子；选落点阶段只能点击或按编号选择合法落点。这样可以避免错误点击或错误按键破坏游戏流程。

新版仍保留 `phase_piece_count`，但主要设置流程集中在 `phase_player_count` 阶段完成。这样既保留旧版阶段结构，又让设置界面更直接。

### 8.2 标准星形棋盘生成

棋盘并不是逐个手写坐标，而是根据每行孔位数量自动生成。

每个孔位保存：

```cpp
struct pointdata
{
    int id;
    int row;
    int col;
    double x;
    double y;
    std::vector<int> nearids;
};
```

横坐标根据当前行长度居中计算，纵坐标根据行号和行间距计算。这样可以保证 121 个孔位形成规整的星形棋盘。

### 8.3 相邻边生成

程序遍历任意两个孔位，计算它们的屏幕距离。如果距离接近一个孔位间距 `map_point_space`，就认为两个孔位相邻，并加入 `nearids`。

这种做法不需要手动列出每个点的邻接表，减少了固定地图数据的维护成本。

### 8.4 跳跃边生成

跳跃边通过“两倍孔位间距 + 中点存在孔位”判断：

1. 遍历两个孔位 A 和 B。
2. 如果 A、B 的距离约等于两个孔位间距，继续判断。
3. 计算 A、B 的中点坐标。
4. 如果中点位置存在孔位 C，则说明可以从 A 跳过 C 到 B。
5. 将 `{ overid = C, toid = B }` 加入 A 的跳跃边列表。

这样可以自动生成六个方向上的跳跃关系。

### 8.5 合法落点计算

`board_get_targets()` 用于计算某个棋子的全部可落点，主要分两部分：

1. 检查所有相邻孔位，空位即可一步移动。
2. 使用队列搜索所有连续跳跃可以到达的位置。

简化逻辑如下：

```text
把起点加入队列
while 队列不空:
    取出当前位置
    枚举当前位置的所有跳跃边
    如果中间孔位有棋子，落点为空，并且落点没有访问过:
        加入合法落点
        加入队列继续搜索
```

这个过程本质上是 BFS 搜索，可以找出一回合内所有连续跳跃可达位置。新版在显示层给前 9 个落点编号，键盘选择落点时使用 `movetargets[number - 1]` 映射到真实孔位，核心搜索算法没有改变。

### 8.6 虚拟局面判断

连续跳跃时，棋子已经从起点移动到了搜索中的当前位置，但真实棋盘数据还没有立即修改。因此代码使用虚拟判断函数处理这种情况：

- 起点在虚拟局面中视为空。
- 当前搜索位置视为有棋子。
- 其他位置按照真实棋盘判断。

这样可以避免连续跳跃搜索时把原起点误判为仍被占用，也能正确判断当前模拟位置。

### 8.7 撤销一步

每次棋子成功移动前，程序会把移动信息保存到 `history`：

```cpp
struct movedata
{
    int playerindex;
    int pieceindex;
    int fromid;
    int toid;
    int orderindex;
};
```

点击“返回上一步”、按 `Esc` 或按 `Delete` 时，程序会根据当前阶段执行不同返回逻辑。若已经产生真实移动，则取出最近一次移动记录，把棋子从 `toid` 放回 `fromid`，并恢复当时的回合顺序下标。若仍处于选择落点阶段，则只取消当前选中棋子，不撤销已经完成的历史移动。

## 9. 变量命名与代码风格

本项目整体采用英文语义命名，并保留大量中文注释，便于课程检查、答辩展示和后续维护。

- 枚举名使用小写单词组合：`phasekind`、`buttonkind`。
- 数据结构使用 `data` 后缀：`boarddata`、`pointdata`、`playerdata`、`piecedata`、`gamestate`。
- 棋盘相关函数使用 `board_` 前缀：`board_build()`、`board_get_targets()`。
- 绘图相关函数使用 `drawer_` 前缀：`drawer_draw()`、`drawer_hit_button()`。
- 游戏流程函数使用 `game_` 前缀：`game_reset()`、`game_process_click()`。
- 键盘操作函数使用 `keyboard_` 前缀：`keyboard_handle_key()`、`keyboard_get_piece_number()`。
- 日志函数使用 `logger_` 前缀：`logger_open()`、`logger_write()`。
- 常量集中放在 `mapdata.h` 和 `keyboard_data.h` 中，便于统一调整界面尺寸、棋盘布局和键盘键值规则。

## 11. 未完成内容与后续可优化方向

当前项目已经可以完成单局跳棋流程，但仍有以下可优化方向：

1. **增加随机或抽签行动顺序**  
   当前行动顺序固定为玩家编号从小到大。后续可以加入开局抽签或随机排序。

2. **增加电脑玩家 AI**  
   当前所有玩家都需要人工或键盘操作。后续可以增加简单 AI，例如优先向目标区靠近、优先连续跳跃、优先进入停车区。

3. **完善标准规则限制**  
   当前重点实现基本移动、跳跃和胜利判断。后续可以根据不同跳棋规则版本，增加对营地进入、离开、占位等细节限制。

4. **增加计时或步数统计**  
   后续可以记录每名玩家移动次数、总用时、平均每步耗时等信息。

5. **增加存档与读档功能**  
   当前只有操作日志，不能保存当前棋局。后续可以把玩家、棋子位置、回合顺序和历史记录保存到文件。

6. **增加音效和动画**  
   当前移动是瞬间完成。后续可以增加棋子移动动画、跳跃动画、胜利动画和按钮音效。

7. **优化界面美术**  
   当前采用基础圆形和矩形绘制，后续可以增加棋盘背景、主题皮肤、棋子图标和更丰富的提示信息。

8. **补充规则测试用例**  
   对相邻边生成、跳跃边生成、连续跳跃搜索、胜利判断、键盘编号映射和撤销功能增加测试，可以降低后续维护风险。
